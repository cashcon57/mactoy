# Mactoy v0.2.0 — Intel Macs + older macOS

v0.1.x was Tahoe-only: `macOS 26.0` minimum, arm64 only. v0.2.0 is the port that makes Mactoy a real cross-Mac tool. The same `Mactoy-0.2.0.dmg` now runs on:

- **Intel Macs** — both slices in one universal binary (`lipo -info` reports `x86_64 arm64`)
- **Apple Silicon** — native arm64, no Rosetta
- **macOS 13.5 Ventura, 14 Sonoma, 15 Sequoia, 26 Tahoe** — one app, one download
- **OCLP-patched hardware** — nothing we do depends on T2 / Secure Enclave / Apple Intelligence

Liquid Glass stays on Tahoe. On 13.5 / 14 / 15 the glass surfaces fall back to a translucent `regularMaterial` card with optional tint overlay — the layout is identical, the material just isn't real glass.

## Why 13.5 and not 13.0

Early Ventura (13.0 – 13.4) shipped multiple `SMAppService` bugs:

- `register()` returned success while `launchd` never loaded the daemon
- BTM / launchd desync survived reboots
- `Operation not permitted` loops on re-register after `unregister`

Apple fixed them by 13.5 / 14.2. We set `LSMinimumSystemVersion 13.5` so Gatekeeper refuses to install on the broken versions rather than shipping and having the helper silently fail.

## What changed

### Architecture

- **`@Observable` → `ObservableObject` + `@Published`.** The `Observation` framework is macOS 14+. `AppState` is now a `@MainActor final class AppState: ObservableObject` so it works on 13.5. Every view swapped `@Environment(AppState.self)` + `@Bindable var state = state` for `@EnvironmentObject var state: AppState`. `$state.foo` bindings come for free via dynamic member lookup.
- **`.mactoyGlass(cornerRadius:tint:interactive:)` view modifier + `MactoyGlassContainer`.** Both live in `Sources/Mactoy/Views/LiquidGlass.swift`. On macOS 26 they call `.glassEffect(...)` / `GlassEffectContainer` directly; on 13.5 – 15 they render `.regularMaterial` inside a rounded shape (`Capsule()` when no radius is specified — not a 999-pt rounded rect, which doesn't actually collapse to a capsule silhouette) with an optional tint overlay and a 0.5-pt primary stroke. Twenty-one `.glassEffect(...)` call sites and two `GlassEffectContainer { }` wraps migrated.
- **`.onChange(of:) { _, new in }` → `{ new in }`** in `HelperAwaitingApprovalSheet`. The two-arg form is macOS 14+; the single-arg form works since macOS 12.
- **`SystemSettingsStrings.loginItemsPane`** helper added. Returns `"Login Items"` on macOS 13/14, `"Login Items & Extensions"` on 15+. Four user-facing strings in `HelperExplainerSheet`, `HelperAwaitingApprovalSheet`, and two `status = .failed(...)` messages in `AppState` use it now — Ventura users no longer see instructions pointing at a pane name that won't exist on their system until Sequoia.

### Build

- **Universal binary opt-in.** `MACTOY_UNIVERSAL=1 ./scripts/build-app.sh release devid` passes `--arch arm64 --arch x86_64` through `swift build` to produce a fat binary in one pass. SwiftPM places the output under `.build/apple/Products/Release/`; `--show-bin-path` returns the right directory automatically. Default builds stay native-arch for local dev speed.
- **Bash 3.2 safe array expansion.** The array that carries `--arch` flags is expanded via `${ARCH_ARGS[@]+"${ARCH_ARGS[@]}"}` so the script works on macOS-shipped `/bin/bash` 3.2 (which otherwise trips `set -u` on an empty `"${arr[@]}"`).

### Docs

- README badges updated: macOS badge is `13.5+` with a new "Universal (arm64 + x86_64)" badge alongside.
- Build-from-source section now correctly lists Xcode 16+ / Swift 6.0+ (Package manifest uses `swift-tools-version: 6.0`, which Swift 5 toolchains can't parse).
- Status section's Liquid Glass line explains the 13.5 – 15 fallback.

## Install

```sh
# Download Mactoy-0.2.0.dmg from the release assets.
open Mactoy-0.2.0.dmg
# Drag Mactoy.app into /Applications.
```

Apple-notarized. If you already have v0.1.4 installed on Tahoe, v0.2.0 drops in without a second Login Items / Full Disk Access round-trip — the helper registration carries over.

## Known limitations

- **No Intel-hardware smoke test.** This release's universal binary is verified static-only (`file`, `lipo -info`). Runtime behaviour on Intel Ventura / Sonoma / Sequoia is being validated via user reports from v0.2.0 onwards. If you hit an issue on Intel, please open a GitHub issue with your macOS version + Mac model.
- **Fallback glass is not real glass.** On 13.5 – 15 the cards use `regularMaterial` with a subtle tint overlay. When **Reduce Transparency** is on, materials fall back to flat opaque colors and the tint becomes faint — acceptable, but different from Tahoe.
