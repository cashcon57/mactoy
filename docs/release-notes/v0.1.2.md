# Mactoy v0.1.2 — Ko-fi + security hardening

## What changed

### Ko-fi support

- **Ko-fi button in the app.** Bottom-left corner of every tab's action bar, using Ko-fi's official dark-on-dark pill asset. Opens `ko-fi.com/cash508287` in your default browser.
- **GitHub "Sponsor" button enabled.** `.github/FUNDING.yml` points at the same Ko-fi page.
- **Ko-fi badge in the README** next to the existing shields.

### Security hardening

These close the highest-impact findings from an internal review of the v0.1.1 codebase.

- **SHA-256 verification of the Ventoy tarball** before the bytes are handed to the privileged helper. The expected hash is pulled from `sha256.txt` published alongside each Ventoy release, so corrupted downloads, middlebox rewrites, and cached-but-stale tarballs all abort the install cleanly. Same-origin assumption (Ventoy's own GitHub account) is documented as a known limitation in [`SECURITY.md`](../../SECURITY.md).
- **Whole-disk name regex.** `InstallPlan.validate` now rejects anything that isn't `^disk[0-9]+$` — slice names (`disk2s1`), path-traversal (`../disk2`), and nonsense (`diskfoo`) can no longer reach `/dev/rdisk*`.
- **Raw-image driver re-probes the disk in the helper.** `RawImageDriver` no longer trusts the `isExternal`/`isRemovable` flags set in the plan; it re-runs `diskutil info` inside the privileged helper and refuses internal disks there, matching what the Ventoy driver already did.
- **Version-string allowlist.** Ventoy version strings are now matched against `[0-9]+(\.[0-9]+){1,3}` before they are interpolated into the GitHub download URL. `../` / newlines / `&&` / query strings in a spoofed version are rejected before URL construction.
- **Tar extraction uses `--no-same-owner --no-same-permissions`**, and happens only after SHA-256 verification — combined defense against zip-slip and root-owned extracts.
- **Pinned minimal environment** (`PATH`, `LC_ALL`, `LANG`) when spawning `diskutil` / `tar` / `newfs_exfat` from the helper.
- **Dropped the unused `com.apple.security.automation.apple-events` entitlement.** The admin prompt uses `/usr/bin/osascript` as a local exec, which doesn't need it.
- 5 new tests covering the validation paths above; total `swift test` is 16/16.

### Docs

- New [`SECURITY.md`](../../SECURITY.md) with Mactoy's threat model and what the alpha does **not** yet defend against — most notably the unauthenticated-root-helper issue, which is targeted for v0.2 (SMAppService migration).

## Not changed

No disk-layout math changed. The existing 12 GPT/MBR/layout unit tests still pass bit-identical against the Python reference implementation.

## Install

```sh
# Download Mactoy-0.1.2.dmg from the release assets.
open Mactoy-0.1.2.dmg
# Drag Mactoy.app into /Applications.
```

No Gatekeeper right-click workaround — the DMG is Apple-notarized.

## Verification

**SHA256 of `Mactoy-0.1.2.dmg`** is published below the release assets. Compare against `shasum -a 256 Mactoy-0.1.2.dmg` locally before running.
