# Mactoy v0.1.1 — UI polish

Second alpha. No disk-writing logic changed — this release is cosmetic + information-architecture fixes on top of v0.1.0.

## What changed

- **App icon.** Mactoy now ships with a real icon — USB stick on a silver-bezeled squircle — rendered in the Dock, Finder, and Get Info. Previous build shipped the default placeholder.
- **Info panels no longer pill-stretched.** The "What this does" blocks on every tab were rendering as capsules because `.glassEffect(.regular)` defaults to `Capsule` shape. Explicit `in: .rect(cornerRadius: 16)` everywhere now.
- **Red warning on Flash Image.** A red-tinted Liquid Glass banner now appears above the drop zone: *"This creates bootable media from a single image. If you want to boot multiple ISOs from one drive, use the Install Ventoy tab, then drop your ISOs onto the Ventoy volume."* Prevents the common mistake of `dd`-ing an ISO when the user actually wants a Ventoy multi-boot stick.
- **Version picker is a real dropdown.** Previously a free-form text field. Now a Picker that fetches the last 20 stable Ventoy releases from the GitHub API on launch and lists them under a **Latest** default. A **Custom…** option reveals a text field so users can still pin an exact tag if Mactoy ever stops updating.
- **README rewritten.** Now leads with a **Why Mactoy exists** section — Ventoy has no macOS installer, the official workarounds (LiveCD + second USB, Linux VM, dual-boot) all need another machine before the five-minute task starts, the native Python gist needs Homebrew `xz` + `sudo` + raw device paths, balenaEtcher only covers single-boot sticks, and the MAS sandbox rules out a polished App Store option. Explains why this gap exists and why the project fills it.

## Not changed

- GPT layout, boot-image writing, exFAT formatting, raw `dd` flashing, `.xz`/`.gz` decompression, disk enumeration, privileged helper handoff. All byte-for-byte the same as v0.1.0 — the unit tests still pass bit-identical against the Python reference.

## Known limitations (unchanged from v0.1.0)

- macOS 26 (Tahoe) or newer.
- **Not yet verified on real hardware by the author.** First-party hardware confirmation is the top priority for v0.1.2.
- No cancel button mid-operation.
- Raw-image flashing loads the whole image into memory.
- Privileged helper is invoked via `osascript` admin prompt; SMAppService daemon is planned for v0.2.

## Install

```sh
# Download Mactoy-0.1.1.dmg from the release assets.
open Mactoy-0.1.1.dmg
# Drag Mactoy.app into /Applications.
```

No Gatekeeper right-click workaround — the DMG is Apple-notarized, so first launch opens normally.

## Verification

**SHA256 of `Mactoy-0.1.1.dmg`** is published below the release assets. Compare against the hash shown by `shasum -a 256 Mactoy-0.1.1.dmg` locally before running.
