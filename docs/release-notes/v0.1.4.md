# Mactoy v0.1.4 — Flash Image + Manage Disk catch up

v0.1.3 fixed Install Ventoy to actually work end-to-end on real hardware. This release applies the same handful of disk-handling fixes to the other two tabs so Flash Image and Manage Disk don't inherit the bugs we just stamped out.

No changes to the Ventoy install path itself. If v0.1.3 worked for you, v0.1.4 will too.

## What changed

### Flash Image

- **Raw-disk fd released before macOS re-probes.** `RawImageDriver` now explicitly `close()`es the `DiskWriter` after `fsync`, then calls `diskutil reloadDisk` + `mountDisk` so the flashed image's filesystem shows up immediately instead of requiring an unplug/replug. Same fix applied to `VentoyDriver` in v0.1.3 — just now applied everywhere we write raw bytes.

### Manage Disk

- **Volume-to-disk binding.** The tab no longer matches any volume named `Ventoy` anywhere on the system. It now confirms the `/Volumes/Ventoy` target's `DeviceIdentifier` actually sits on the currently-selected BSD disk via `diskutil info`. If you had two Ventoy sticks plugged in, you could previously have been editing the wrong one.
- **ISO list refreshes on disk switch.** `.id(volumeURL)` forces the list to rebuild when you pick a different disk from the sidebar, instead of showing stale items from the previous drive.
- **Errors surface.** Copy and delete actions used to swallow failures with `try?`. Both now raise an `NSAlert` with the underlying error text.
- **Refresh button** and an `N ISOs on this drive` header added for clarity.

### Docs

- README status section is now **v0.1.3 alpha** with the **"Verified on real hardware"** line checked, and the outdated `osascript` references in the architecture + security sections are replaced with the XPC / `SMAppService` flow that actually ships.

## Install

```sh
# Download Mactoy-0.1.4.dmg from the release assets.
open Mactoy-0.1.4.dmg
# Drag Mactoy.app into /Applications.
```

Apple-notarized. If you already have v0.1.3 installed, the helper is registered and approved — v0.1.4 drops in and runs without a second Login Items / Full Disk Access round-trip.
