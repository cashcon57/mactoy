# Mactoy v0.1.0 — alpha

First public release. Mactoy is a native SwiftUI app for macOS 26+ that installs Ventoy onto a USB drive (or flashes any raw image) without needing a Linux VM or booting the Ventoy LiveCD.

## What works

- **Install Ventoy** — downloads the latest (or pinned) Ventoy release from GitHub, wipes the selected external USB drive, writes a Ventoy-compatible GPT + protective MBR + boot images, formats the data partition as exFAT.
- **Flash raw image** — drag an `.iso`, `.img`, `.img.xz`, or `.img.gz` and write it to a USB drive with a progress bar.
- **Manage existing Ventoy drive** — list / add / remove ISO files on a mounted Ventoy volume.
- **Liquid Glass UI** — sidebar disk picker, mode tabs, action bar all use macOS 26's new glass material.

## What's cross-validated

The GPT layout math is a direct port of the Python proof-of-concept that successfully flashed a Ventoy drive on this machine. Unit tests confirm that for the same disk size, Swift produces bit-identical partition boundaries, CRCs, GUIDs, and boot-sector patches as the Python reference:

```text
✔ 124GB stick matches Python reference
✔ basic data GUID matches GPT wire format
✔ Ventoy partition entry is 128 bytes and starts with type GUID
✔ header is 512 bytes, starts with EFI PART, has valid CRC
✔ signature + protective partition type
✔ part2 starts on 8-sector boundary (across 4000 disk sizes)
✔ CRC32 known vectors
```

12/12 tests pass.

## Signing status

- **Signed:** Developer ID Application "Clayton Conway (MUQ3H79Y4N)" with hardened runtime.
- **Notarized:** not yet. Apple's notarization service needs an Apple ID + app-specific password which weren't available at build time. Notarization is a one-command follow-up (`xcrun notarytool submit Mactoy-0.1.0.dmg --apple-id <you> --team-id MUQ3H79Y4N --password <app-specific>`) and will be completed for v0.1.1.
- **First-launch experience:** because the build is signed-but-not-notarized, Gatekeeper will prompt on first launch. Right-click → Open once, or `xattr -cr /Applications/Mactoy.app` from Terminal. Subsequent launches are silent.

## What's NOT yet shipped (on the v0.2 roadmap)

- **Apple notarization** (see Signing status above).
- **SMAppService privileged helper daemon.** v0.1 uses `osascript` admin prompts — one auth per install. Functional, just not as slick as a persistent daemon.
- **Homebrew cask.** Coming with v0.2 once the build is signed + notarized.
- **Real-hardware confirmation by maintainer.** The code that does the writing is a line-by-line port of a Python script we tested on a 124 GB USB and successfully booted. The Swift port is verified by cross-validation tests. First-party hardware sign-off happens in v0.1.1.

## Install

```sh
# Download Mactoy-0.1.0.dmg from the release assets.
# Open the DMG, drag Mactoy.app to Applications.
xattr -cr /Applications/Mactoy.app
open /Applications/Mactoy.app
```

## Known limitations

- **macOS 26+ only.** Liquid Glass APIs require this. No fallback to older macOS in v0.1.
- **NTFS format not implemented.** Only exFAT for the Ventoy data partition.
- **No cancel button.** Progress bar updates mid-install, but there's no "abort" action yet.
- **Raw-flash is not streaming.** Entire image is loaded into memory before write starts; for very large images (say, macOS installer sizes), use `dd` directly in v0.1.
